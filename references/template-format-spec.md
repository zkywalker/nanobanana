# Template Format Spec

Authoritative reference for template.md structure, repo layout, and sample image requirements. Read this when you need detailed field definitions beyond what `template-system.md` covers.

## Repo Structure

A template repo can contain one or multiple templates:

**Single-template repo** (`user/nanobanana-cyberpunk`):
```
/
├── template.md              # Required: template definition
├── samples/                 # Required: at least 1 sample image
│   ├── sample-01.jpg
│   └── sample-02.png
├── README.md                # Optional
└── LICENSE                  # Optional
```

**Multi-template repo** (`user/nanobanana-templates`):
```
/
├── bananahub.json           # Required: declares multi-template repo
├── cyberpunk-city/
│   ├── template.md
│   └── samples/
│       └── sample-01.jpg
├── neon-portrait/
│   ├── template.md
│   └── samples/
│       └── sample-01.jpg
└── README.md
```

**`bananahub.json`** (multi-template repo manifest):
```json
{
  "type": "multi",
  "templates": ["cyberpunk-city", "neon-portrait"]
}
```

**Repo type detection** (by CLI):
1. Root `template.md` exists → single template
2. Root `bananahub.json` exists → multi template
3. Neither → error

## template.md Complete Field Reference

```yaml
---
# === Identity ===
id: cyberpunk-city                    # Unique ID (lowercase, hyphens, 3-50 chars). Auto-derived from directory name if omitted
title: 赛博朋克城市夜景               # Display title (Chinese)
title_en: Cyberpunk City Nightscape   # Display title (English)
description: 一键生成赛博朋克风格的城市夜景  # Short description for listing (< 50 chars)
author: github-username               # GitHub username of template author
version: 1.0.0                        # Semver

# === Generation Config ===
profile: photo                        # Target profile: photo/illustration/diagram/text-heavy/minimal/sticker/3d/product/concept-art/general
aspect: "16:9"                        # Recommended aspect ratio
models:                               # Tested models with quality ratings
  - name: gemini-3-pro-image-preview
    quality: best                     # best / good / ok
  - name: gemini-2.0-flash-preview-image-generation
    quality: good

# === Discovery ===
tags: [赛博朋克, 城市, 夜景, 科幻, cyberpunk, neon, city]  # Bilingual, ≥ 3 tags
difficulty: beginner                  # beginner / intermediate / advanced
category: wallpaper                   # Optional: wallpaper / social-media / e-commerce / education / marketing / art / other

# === Sample Images ===
samples:
  - file: samples/sample-01.jpg
    model: gemini-3-pro-image-preview
    prompt: "The full English prompt used to generate this sample"
    aspect: "16:9"

# === Metadata ===
created: 2026-03-24
updated: 2026-03-24
---
```

### Required vs Optional Fields

| Field | Required | Used By |
|-------|----------|---------|
| `id` | Yes (or derived from dirname) | CLI install, listing, matching |
| `title` | Yes | Listing, Hub display |
| `title_en` | Yes | Hub display, English search |
| `description` | Recommended | Hub cards, listing |
| `author` | Yes | Attribution, Hub profiles |
| `version` | Yes | Update detection |
| `profile` | Yes | Grouping, profile alignment |
| `aspect` | Yes | Default generation parameter |
| `models` | Yes (≥ 1) | Model selection, Hub display |
| `tags` | Yes (≥ 3) | Auto-matching, search |
| `difficulty` | Yes | Listing, filtering |
| `category` | Optional | Hub filtering |
| `samples` | Yes (≥ 1) | Hub preview, quality proof |
| `created` / `updated` | Yes | Sorting, freshness |

## Body Structure

The body MUST follow this exact section order:

```markdown
## Prompt Template
[code block with {{variable|default}} slots]

## Variables
[pipe table: Variable | Default | Description]

## Tips
[bullet list of specific, actionable tips]
```

See `template-system.md` → Body Authoring Constraints for quality rules.

## Variable Syntax

`{{name|default}}` — double braces, snake_case name, pipe separator, fallback value.

| Pattern | Meaning |
|---------|---------|
| `{{scene\|rain-soaked alley}}` | Variable with default |
| `{{subject}}` | Variable without default (must be provided) |

Rules:
- Names: snake_case, 2-30 chars
- Defaults: complete phrases/sentences (not keyword fragments)
- 3-6 variables per template

## Sample Image Requirements

| Rule | Value |
|------|-------|
| Minimum count | 1 per template |
| Format | JPG or PNG |
| Max size | 2MB per image |
| Naming | `sample-{model-short}-{nn}.png`, e.g. `sample-3.1-flash-01.png`, `sample-3-pro-01.png` |
| Metadata | Each sample MUST have `model` and `prompt` in frontmatter |
| Purpose | Show what the template produces; Hub displays as preview cards |

## .source.json (Install Provenance)

Auto-generated by CLI on install, stored alongside template:

```json
{
  "repo": "user-a/nanobanana-cyberpunk",
  "ref": "main",
  "sha": "abc123...",
  "installed_at": "2026-03-24T10:00:00Z",
  "version": "1.0.0",
  "cli_version": "0.1.0"
}
```

  "version": "1.0.0",
