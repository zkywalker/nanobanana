# Local Asset Style Consistency Workflow

A BananaHub workflow template for building a consistent asset pack from local images. It is meant for agent runs where one approved anchor style should be propagated across additional images without changing locked composition, text, or subject identity.

## Install

```bash
npx bananahub add bananahub-ai/bananahub-skill/asset-style-consistency-pack
```

## Verified Models

- `gemini-3-pro-image-preview` — verified with `samples/sample-3-pro-01.png` and `samples/sample-3-pro-02.png` using the bundled local Minecraft house screenshot as the input image

## Supported Models

- `gemini-3-pro-image-preview` — strongest option when structural preservation and finish consistency both matter
- `gemini-3.1-flash-image-preview` — useful for quicker style-normalization experiments before the final pass

## Sample Outputs

| File | Model | Prompt / Variant |
|---|---|---|
| `samples/sample-3-pro-01.png` | `gemini-3-pro-image-preview` | Style-normalized architectural concept illustration generated by editing the bundled local input image while preserving the building layout, pool, fountain, trees, and camera angle |
| `samples/sample-3-pro-02.png` | `gemini-3-pro-image-preview` | Premium dusk visualization generated by editing the same local input image while preserving layout and composition but shifting the whole scene into an evening-render family |
